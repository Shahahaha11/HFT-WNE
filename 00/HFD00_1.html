Introduction

To use Quarto/Markdown you should install the quarto program. And to use it in VS Code you need to have the Quarto VS Code Extension.

In python it would also require installing the following libraries most commonly required by quarto: jupyter, ipykernel, nbformat, nbclient, pyyaml.

Once the document is ready to generate a result just click Preview button on right hand side of the top bar of VS Code.

The text inside Quarto documents is written in Markdown. Markdown is a simple set of conventions for adding formatting to plain text.

This starts a new subchapter

Your document can be organized in (sub)capters. One can also use itemization:

  • this is item 1,
  • there might be a second one,
    • subitems are possible as well
    • there might be more of them,
  • in fact as many as you wish.

Numbered lists are also possible:

  1. this is item number 1 again,
  2. there might be a second one,
    • subitems are possible as well
    • there might be more of them,
      • subsubitems are also allowed
  3. in fact as many as you wish.

Second subchapter

If you want to emphasize text – use a star * for italics or double star ** for bolded text (slash \ turns off a special meaning of characters).

For more details please check the Markdown basics, Quarto guide or Tutorials.

Subchapter no 3 – code chunks

But the most interesting feature is the possibility of creating a dynamic document – mix text with python code and python results – this is done in so called code chunks – see below for examples.

Example 1

This chunk prints the code, evaluates it and prints the results

import numpy as np
import pandas as pd

# Generate 1000 random values from a standard normal distribution
a = np.random.normal(size=1000)

# Use pandas to get a summary
summary = pd.Series(a).describe()
print(summary)

Example 2

By using an additional chunk option #|eval: false you can for example see just the code and not the results (the code is printed, but not executed/evaluated):

import numpy as np
import pandas as pd

# Generate 1000 random values from a standard normal distribution
a = np.random.normal(size=1000)

# Use pandas to get a summary
summary = pd.Series(a).describe()
print(summary)

One can also refer to results of Python objects inline in the text.

Inline expressions should be confined to simple values that you have pre-computed within normal code cells (rather than function calls that do non-trivial work). This is because the protocol used for inline expressions is not compatible with some Python libraries (especially those that use multi-threading or multi-processing).

For example: on 2025-09-29 Apple quotations started with the price of 254.56$ to finish a day on a level of 254.43$ with the volume of trading equal to 40127700 units.

Dollars in Quarto/Markdown

We had to mask a dollar ($) sign in the above examples as it has a special meaning in Markdown – very similar to LaTeX – it can be used to indicate a mathematical formulea, for example: \(x_{1+s}^{2*d}\), \(\frac{y^2}{x_{1t}}\), \(y_t = \alpha_0 + \phi_1 x_{1,t} + \epsilon_t\).
